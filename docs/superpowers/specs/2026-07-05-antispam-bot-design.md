# Telegram-бот для автоудаления повторяющегося спама

Дата: 2026-07-05

## Проблема

В групповой чат постоянно шлют спам с одинаковым текстом (разные аккаунты,
один и тот же текст). Нужен бот, которого добавляют в чат админом, и который:

- позволяет админу реплаем пометить сообщение как спам командой `/add_to_spam`;
- сохраняет точный текст спама в базу, отдельно по каждому чату;
- при появлении **байт-в-байт** идентичного текста от кого угодно —
  удаляет сообщение и банит автора в этом чате;
- по стилю общения ориентируется на @Admitix_Bot.

## Архитектура

- **Python 3.11+, aiogram 3.x**, long polling. Не нужен домен/HTTPS/вебхук —
  бот сам опрашивает Telegram, запускается локально или на любом VPS одной
  командой.
- **Хранилище:** SQLite (`aiosqlite`), один файл `bot.db`. Разделение по чатам —
  через колонку `chat_id`, не через отдельные файлы БД.
- **Конфиг:** `.env` (`BOT_TOKEN`), не коммитится, есть `.env.example`.

## Модель данных

### `spam_texts`
| поле | тип | описание |
|---|---|---|
| id | INTEGER PK AUTOINCREMENT | |
| chat_id | INTEGER | id чата в Telegram |
| text | TEXT | точный текст спама |
| added_by | INTEGER | user_id админа, добавившего запись |
| added_at | TIMESTAMP | |

Ограничение `UNIQUE(chat_id, text)` — не даёт задвоить запись.

### `chat_owners`
| поле | тип | описание |
|---|---|---|
| chat_id | INTEGER PK | |
| inviter_user_id | INTEGER | кто добавил бота в чат (`my_chat_member.from`) |
| updated_at | TIMESTAMP | |

Используется для ЛС-уведомлений об автобанах.

## Команды (только для админов/владельца чата)

Проверка прав — через `bot.get_chat_administrators(chat_id)`, статус
`administrator` или `creator`. Результат кэшируется на ~5 минут на чат, чтобы
не дёргать API на каждое сообщение при автомодерации. Не-админам команды
молча игнорируются (без подсказок о существовании).

### `/add_to_spam`
Только как reply на сообщение.

1. Проверить, что вызвавший — админ/владелец. Если нет — игнор.
2. Проверить, что у реплайнутого сообщения есть текст. Если нет —
   ответить «⚠️ В этом сообщении нет текста для сохранения».
3. Сохранить `(chat_id, text)` в `spam_texts` (если уже есть такая запись —
   не дублировать, продолжить как обычно).
4. В зависимости от автора реплайнутого сообщения:
   - **обычный пользователь** → удалить сообщение + забанить автора в чате
     навсегда (`banChatMember` без `until_date`).
   - **админ/владелец чата** → удалить только сообщение, **не банить**.
   - **сам бот** → не удалять/не банить, текст всё равно сохраняется.
5. Ответить в чат коротким подтверждением:
   - «✅ Добавлено в спам-лист (#N). Автор забанен.»
   - «✅ Добавлено в спам-лист (#N). Сообщение удалено (автор — админ, бан
     пропущен).»
   - Если удаление/бан не удались из-за нехватки прав бота — «⚠️ Текст
     сохранён, но не удалось удалить/забанить — проверьте права бота
     (Delete messages, Ban users)».

### `/list_spam`
Админ-онли. Выводит нумерованный список сохранённых текстов чата: `id` +
превью текста (обрезка ~80 символов).

### `/remove_from_spam <id>`
Админ-онли. Удаляет запись `id` из `spam_texts` для этого чата. Если `id` не
найден в этом чате — сообщение об ошибке.

## Автомодерация

На каждое **не-командное текстовое** сообщение в группе, где есть бот:

1. Если отправитель — админ/владелец чата или сам бот → пропустить полностью,
   никаких проверок и действий.
2. Сравнить `message.text` **байт-в-байт** (регистр, пробелы, переносы строк
   — всё учитывается) со всеми записями `spam_texts` этого `chat_id`.
   Сравнение идёт по plain-тексту — URL из text_link entities в `.text` не
   попадают, поэтому на совпадение не влияют.
3. При точном совпадении:
   - удалить сообщение;
   - забанить автора в этом чате навсегда;
   - **ничего не писать в группу**;
   - отправить ЛС владельцу чата (`chat_owners.inviter_user_id`):
     «🛑 В чате «{название чата}» удалено спам-сообщение от {@username или
     имя}, автор забанен.» Если ЛС не доставляется (пользователь не жал Start
     у бота) — тихо залогировать и не повторять попытку.

**Определение владельца чата:** хендлер на апдейт `my_chat_member` — когда
статус бота меняется на `member`/`administrator` (бота только что добавили),
записать/обновить `chat_owners.inviter_user_id = update.from_user.id`.

## Требования к правам бота

Бота нужно добавить в чат админом с правами **Delete messages** и
**Ban users** — без этого автомодерация и `/add_to_spam` не смогут
удалять/банить. Это будет описано в README как обязательный шаг настройки.

## Обработка ошибок

- Нет прав на удаление/бан → Telegram кидает ошибку, бот логирует её;
  при `/add_to_spam` дополнительно предупреждает в чате (см. выше).
- Реплай без текста → предупреждение, запись не создаётся.
- Команда не от админа → молчаливый игнор.
- ЛС владельцу чата не доставлено → тихий лог, без ретраев и без сообщений
  в группу.

## Деплой

- `requirements.txt`: `aiogram`, `aiosqlite`, `python-dotenv`.
- `.env.example` с `BOT_TOKEN=`.
- `README.md`: как получить токен у BotFather, как добавить бота в чат
  админом с нужными правами, как запустить (`python -m bot`).
- SQLite-файл и таблицы создаются автоматически при первом запуске
  (`CREATE TABLE IF NOT EXISTS`).

## Тесты

- Юнит-тесты на точное сравнение текста (регистр/пробелы/переносы строк).

  `pytest` + временная SQLite-база.
- Хендлеры aiogram тестируются через мок `Bot`/`Message`.
- Полный e2e — ручная проверка в тестовой Telegram-группе (обязательна,
  библиотечные моки такое поведение Telegram не гарантируют).
